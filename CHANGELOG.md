- 纯 SSH 无桌面环境下，浏览器窗口通常无法直接显示

## F0.0.9

### 修复
- 修复 Windows 下点击“更新脚本”后，重启命令被错误解析导致找不到文件/卡死的问题
- 不再依赖 `cmd / start` 的字符串拼接重启方式，避免路径中包含空格或参数时出错

### 优化
- Windows 改为通过独立 Python 辅助进程延迟拉起新实例
- 自更新完成后，旧进程会更干净地退出，新进程重新启动更稳定

---

## F0.0.8

### 调整
- 抢购时间的锁定时机从“确认购物车后”提前到“确认登录后”
- 这样在进入购物车与结算流程前，目标时间就已经固定，避免后续阶段继续修改导致执行时机混乱

### 说明
- 当前规则变更为：
  - **点击开始抢购后**：仍可修改时间
  - **点击确认登录后**：立即锁定时间
  - **确认购物车后**：保持既有锁定状态，不再重复变更

---

## F0.0.7

### 修复
- 修复“更新脚本”执行完成后，当前实例重启可能卡死的问题
- 避免在请求线程中直接 `execv` 重启自身，改为先拉起新的独立进程，再安全退出旧进程

### 优化

- 从这个版本开始，每次功能更新都同步维护 `CHANGELOG.md`，并准备发布 Release 产物

## F0.0.2

### 新增
- 页面增加 **更新脚本** 按钮
- 点击后可自动执行：
  - `git pull --rebase --autostash origin main`
  - `pip install -r requirements.txt`
  - 自动重启当前实例

### 修复
- 修复多开场景下实时日志可能遗漏“抢购成功”状态的问题
- 在 SSE 丢事件时，前端会通过状态轮询自动补齐日志
- 当任务已经成功结束时，前端会自动同步到“抢购完成”阶段并启用关闭浏览器按钮

### 优化
- 扩大任务日志缓冲区，减少高频操作时日志被过快挤掉的概率
- 任务创建计数增加线程锁，降低并发场景下的状态竞争风险
