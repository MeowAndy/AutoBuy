# 更新日志

## F0.0.6

### 修复
- 修复在确认购物车后仍然沿用旧抢购时间的问题
- 现在会以最新同步到后端的 `target_time` 为准进入等待抢购阶段

### 优化
- `SeckillWorker` 在启动和等待阶段统一使用实例内的最新目标时间，避免出现前端已改时间但执行层仍读旧值的情况

---

## F0.0.5

### 新增
- 抢购开始后，在 **确认购物车之前** 允许继续修改抢购时间
- 当前端修改时间时，后端会同步更新任务的 `target_time`

### 优化
- 只有在点击 **确认购物车** 之后，才锁定时间输入框
- 更贴近真实使用场景，避免忘记先设置时间又被迫重开任务

---

## F0.0.4

### 新增
- `start.bat` 在端口被占用时，支持从当前端口开始自动递增寻找空闲端口（如 `5000 -> 5001 -> 5002`）
- 启动成功后自动在默认浏览器打开对应地址，无需再手动复制链接

### 优化
- 保留手动输入端口与命令行传参启动方式，自动探测逻辑作为补充，不破坏原有习惯

---

## F0.0.3

### 新增
- `start.bat` 支持在启动时提示输入端口
- 双击同一个 `start.bat` 即可按需输入不同端口运行多个实例

### 优化
- 如果直接回车，默认仍使用 `5000` 作为启动端口
- README 补充了端口输入式启动说明

---

## F0.0.2

### 新增
- 页面增加 **更新脚本** 按钮
- 点击后可自动执行：
  - `git pull --rebase --autostash origin main`
  - `pip install -r requirements.txt`
  - 自动重启当前实例

### 修复
- 修复多开场景下实时日志可能遗漏“抢购成功”状态的问题
- 在 SSE 丢事件时，前端会通过状态轮询自动补齐日志
- 当任务已经成功结束时，前端会自动同步到“抢购完成”阶段并启用关闭浏览器按钮

### 优化
- 扩大任务日志缓冲区，减少高频操作时日志被过快挤掉的概率
- 任务创建计数增加线程锁，降低并发场景下的状态竞争风险

---

## F0.0.1

### 新增
- 支持通过 `python app.py --port <端口>` 启动不同实例
- 支持通过环境变量 `PORT` / `HOST` / `DEBUG` 控制启动参数
- `start.bat` 支持命令行传参指定端口（如 `start.bat 5001`）

### 优化
- 为一台电脑多开多个 Web 实例提供基础能力
- README 与帮助页面同步更新端口化说明

### 说明
- 本版本主要解决“一台本地电脑无法多端口多开”的问题，为后续多实例隔离和稳定性增强打基础
